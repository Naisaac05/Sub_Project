## 공통

### 접근 제어

- `/admin/users/**` 경로는 **ADMIN 역할**만 접근. 백엔드는 `SecurityConfig.java:49` 의 `/api/admin/**` 패턴으로 이미 가드됨.
- 프런트 가드: `app/admin/layout.tsx` 에서 이미 ADMIN 체크 → 이 하위 라우트는 자동 보호.

### 라우팅

| 경로 | 화면 |
|------|------|
| `/admin/users` | 회원 목록 (탭: 전체 / 멘티 / 멘토 / 관리자) |
| `/admin/users/[id]` | 회원 상세 + 역할 변경 |

### API 매핑 (신규 백엔드 작업 필요)

| 동작 | HTTP | 엔드포인트 | body | 현황 |
|------|------|-----------|------|------|
| 목록 | GET | `/api/admin/users?role={MENTEE\|MENTOR\|ADMIN\|ALL}&q={검색어}&page=&size=` | — | ❌ 없음, 신규 |
| 상세 | GET | `/api/admin/users/{id}` | — | ❌ 없음, 신규 |
| 역할 변경 | PUT | `/api/admin/users/{id}/role` | `{role: "MENTEE\|MENTOR\|ADMIN"}` | ❌ 없음, 신규 |

> ⚠ **갭**: `UserController` 는 `/me` 2개 엔드포인트만 존재 (`UserController.java:25, 33`). 관리자용은 **신규 `AdminUserController` 작성 필요**. ROADMAP Phase 6 에 정의된 `GET /api/admin/users`, `PUT /api/admin/users/{id}/role` 와 동일.

### 역할 변경 정책

- MENTEE ↔ MENTOR 간 직접 변경은 **금지** (멘토는 승인 플로우 거쳐야 함 → `/admin/mentor` 로 안내).
- 가능한 변경:
  - MENTEE → ADMIN (스태프 지정)
  - MENTOR → ADMIN
  - ADMIN → MENTEE (권한 회수)
  - ADMIN → MENTOR (권한 회수 + 기존 MentorProfile 있는 경우)
- **자기 자신 강등 금지** (현재 로그인 ADMIN이 자신을 MENTEE/MENTOR로 바꾸려 하면 차단).

## 1. `/admin/users` — 회원 목록

```
┌────────────────────────────────────────────────────────────────────┐
│ 회원 관리                                                          │  h1
│ 전체 회원을 조회하고 역할을 관리합니다.                             │  muted
└────────────────────────────────────────────────────────────────────┘

┌────────────────────────────────────────────────────────────────────┐
│ [ 전체 ]  [ 멘티 (132) ]  [ 멘토 (21) ]  [ 관리자 (3) ]             │  Tabs
│                                  🔍 이름·이메일 검색  [_______]     │  Input (debounced)
└────────────────────────────────────────────────────────────────────┘

┌────────────────────────────────────────────────────────────────────┐
│ 이름       이메일               역할    가입일       최종 로그인     │  TableHeader
├────────────────────────────────────────────────────────────────────┤
│ 김멘티     mentee1@x.com        🟢멘티  2026-03-02  04-20 14:22     │  TableRow × N
│                                                  [상세 ›]           │
├────────────────────────────────────────────────────────────────────┤
│ 이멘토     mentor2@x.com        🔵멘토  2026-03-10  04-21 09:10     │
│                                                  [상세 ›]           │
├────────────────────────────────────────────────────────────────────┤
│ admin      admin@devmatch       🟣관리자 2026-02-01  04-22 10:00     │
│                                                  [상세 ›]           │
└────────────────────────────────────────────────────────────────────┘

← 이전   [1] 2 3 ...   다음 →                                          Pagination
```

### 필터 & 검색

- 탭: `ALL` / `MENTEE` / `MENTOR` / `ADMIN` — URL 쿼리 `?role=MENTOR` 동기화.
- 검색 인풋: 300ms debounce → 서버 `q` 파라미터로 전송 (이름·이메일 LIKE).
- 페이지네이션: 서버 사이드 (size=20 기본). Phase 6 공통 모듈 사용.

### 최종 로그인 표기

- `User` 엔티티에 `lastLoginAt` 필드 **없음** — 표시하려면 추가 필요.
- **Phase II에서는 컬럼 생략** 또는 `updatedAt` 임시 노출 + "정확한 로그인 시각 아님" 주석. 결정 필요 (§ 결정 사항).

### 빈 상태 / 로딩 / 에러

| 상황 | 렌더 |
|------|------|
| 로딩 | `Skeleton` 5줄 |
| 에러 | `Alert variant="destructive"` + 재시도 |
| 빈 목록 | "해당 역할의 회원이 없습니다" (muted) |

### shadcn 컴포넌트 매핑

| UI 요소 | shadcn 컴포넌트 | 비고 |
|---------|----------------|------|
| 역할 탭 | `Tabs` | Phase I에서 이미 설치 |
| 검색 인풋 | `Input` + `Search` 아이콘 | 300ms debounce 커스텀 |
| 표 | `Table` | Phase I에서 이미 설치 |
| 역할 배지 | `Badge` | MENTEE=emerald, MENTOR=sky, ADMIN=violet |
| 상세 버튼 | `Button variant="ghost"` + `ChevronRight` | |
| 페이지네이션 | `Pagination` | **`npx shadcn@latest add pagination` 필요** |

## 2. `/admin/users/[id]` — 회원 상세 + 역할 변경

```
┌────────────────────────────────────────────────────────────────────┐
│ [← 목록]   김멘티 · mentee1@x.com                   🟢 멘티         │  Header + Badge
│            가입일 2026-03-02 · ID 42                               │  muted
└────────────────────────────────────────────────────────────────────┘

┌─ A. 기본 정보 ─────────────────────────────────────────────────────┐
│ 이름     김멘티                                                    │  Card
│ 이메일   mentee1@x.com                                             │
│ 가입 경로 email (또는 google / kakao)                              │
│ 가입일   2026-03-02 10:15                                          │
│ 최종 수정 2026-04-18 16:02                                         │
└────────────────────────────────────────────────────────────────────┘

┌─ B. 역할 관리 ─────────────────────────────────────────────────────┐
│ 현재 역할   🟢 멘티                                                │
│ 변경       [ ▾ 선택 ] → [ 관리자 ]        [ 변경 확정 ]            │  Select + Button
│                                                                    │
│ ⓘ 멘티 ↔ 멘토 직접 변경은 불가능합니다.                            │  Alert info
│   멘토 등록은 "멘토 심사" 에서 승인 플로우로 진행해 주세요.         │
└────────────────────────────────────────────────────────────────────┘

┌─ C. 연관 활동 요약 ────────────────────────────────────────────────┐
│ 결제 내역       3건 (확정 2, 취소 1)     [결제 관리에서 보기 ›]    │
│ 작성 게시글      12건                    [게시물 관리에서 보기 ›]    │
│ (멘토인 경우) 멘토 프로필   🔗 상세 →                              │
└────────────────────────────────────────────────────────────────────┘
```

### 역할 변경 동작

1. `Select` 드롭다운 옵션은 **정책상 가능한 것만** 동적 필터링:
   - 현재 MENTEE → ADMIN만 선택 가능
   - 현재 MENTOR → ADMIN만 선택 가능
   - 현재 ADMIN → MENTEE / MENTOR 선택 가능 (자기 자신이 아닐 때)
2. 변경 확정 클릭 → **확인 모달** (Dialog) 오픈.
3. 확인 → `PUT /api/admin/users/{id}/role` → 성공 시 페이지 리로드 + `toast.success`.
4. 본인 계정 강등 시도 → 프론트에서 미리 차단 (모달 뜨지 않음) + 안내 문구.

### shadcn 컴포넌트 매핑

| UI 요소 | shadcn 컴포넌트 | 비고 |
|---------|----------------|------|
| 기본 정보 카드 | `Card` + `<dl>` | Phase I 동일 패턴 |
| 역할 Select | `Select` + `SelectTrigger/Content/Item` | **`npx shadcn@latest add select` 필요** |
| 확정 버튼 | `Button variant="default"` | |
| 안내 Alert | `Alert variant="default"` | |
| 확인 모달 | `Dialog` | Phase I에서 이미 설치 |

## 결정 필요 사항

1. **최종 로그인 시각 표시** — `User.lastLoginAt` 필드 추가 여부
   - (a) 추가하고 `AuthService` 에서 로그인 시 갱신
   - (b) Phase II 에서는 컬럼 생략, `updatedAt` 도 노출 안 함
   - (c) `updatedAt` 임시 노출 + 안내 툴팁
2. **페이지네이션** — Phase I 에서는 제외됐으나 회원은 규모가 크므로 **서버 페이징 포함**이 기본. 동의 필요.
3. **검색 스펙** — 이름+이메일 LIKE 외에 가입일 range, 역할 다중 선택까지 확장할지.
4. **역할 변경 이력 테이블** — 감사 로그를 위한 `UserRoleHistory` 테이블 신설 여부. (멘토는 이미 `MentorProfileHistory` 있음.)
