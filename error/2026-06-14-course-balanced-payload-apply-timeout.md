# 코스 균등 payload 적용 프로세스 시간 초과

- 발생 일시: 2026-06-14
- 영역: AI / RAG card batch
- 심각도: medium

## 증상

10개 카드의 payload를 카드별 검색 평가와 함께 적용하던 명령이 120초 제한을 넘겼다. 9개 카드 파일이 기록된 뒤 마지막 카드 처리 중 프로세스가 종료되어 최종 적용 보고서가 생성되지 않았다.

## 원인

`ai/app/scripts/apply_payload_batch_v217.py`는 카드마다 production/content 검색 평가를 다시 실행한다. 10개 카드 평가 시간이 명령 실행 제한보다 길어 배치 완료 전에 외부 프로세스가 종료됐다.

## 해결 방법

새 배치 백업 `ai/app/knowledge/concepts_v2_backups/course_balanced_ready10_20260614T175415939017`에서 대상 10개만 복원했다. 이후 동일 배치를 충분한 명령 제한으로 다시 실행한다.

## 재발 방지·메모

카드별 retrieval 평가를 포함한 10개 이상 적용 배치는 최소 10분의 실행 제한을 사용한다. 최종 보고서가 없는 경우 부분 성공으로 간주하지 않고 해당 배치의 새 백업에서 대상 카드만 복원한다.
