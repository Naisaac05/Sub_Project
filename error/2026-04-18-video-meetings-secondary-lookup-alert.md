# 화상회의 페이지 진입 시 보조 조회 실패로 알림이 두 번 뜨는 문제

- 발생일: 2026-04-18
- 영역: frontend
- 심각도: medium

## 증상
멘토링 세션을 만든 직후 `화상회의` 페이지로 이동하면 `화상회의 정보를 불러오지 못했습니다.` 알림이 뜨고, 개발 환경에서는 같은 알림을 두 번 눌러야 했습니다.  
그 뒤에는 세션 자체의 `meetLink` 기반 카드가 보여서 실제로는 회의가 하나 생긴 것처럼 보였습니다.

## 원인
화상회의 페이지는 먼저 세션 목록을 읽은 뒤, 각 세션에 대응하는 `video_meetings` 레코드를 추가로 조회합니다.  
이 보조 조회가 실패하면 세션 목록은 이미 정상인데도 전체 `fetchData` 를 실패 처리해 `alert()` 를 띄우고 있었고, 개발 모드의 effect 재실행 때문에 같은 알림이 반복됐습니다.

## 해결 방법
- `frontend/src/app/lms/(dashboard)/video-meetings/page.tsx:108` 에서 세션별 화상회의 상세 조회 실패를 전체 페이지 실패로 승격하지 않고 `null` 로 흡수하도록 변경했습니다.
- 같은 파일에서 세션 목록 조회 자체가 실패한 경우에만 페이지 메시지로 안내하도록 유지해, 보조 조회 문제로 브라우저 알림이 뜨지 않게 했습니다.

## 재발 방지 / 메모
- 목록 조회 후 부가 정보를 N건 추가 조회하는 화면에서는, 각 항목 단위 실패를 전체 화면 실패로 올릴지 먼저 구분해야 합니다.
- 개발 모드에서는 effect 재실행으로 같은 증상이 두 번 보일 수 있으니, `alert()` 같은 전역 UI는 가능한 한 피하고 화면 상태로 표현하는 편이 안전합니다.
