## 1. 현재 DevMatch 디자인 시스템 분석

### 1.1 현재 사용 중인 디자인 언어

| 요소 | 현재 값 | 비고 |
|------|--------|------|
| **테마** | 다크 (로그인/대시보드) + 라이트 (콘텐츠) | 하이브리드 |
| **배경** | `#0a0e1a` (다크), `white`/`gray-50` (라이트) | |
| **주 색상** | `blue-500` ~ `blue-600` | Tailwind blue |
| **보조 색상** | `cyan-400`, `violet-500` | 그라데이션 활용 |
| **카드** | `glass-card` (backdrop-blur, border-white/5) | 글래스모피즘 |
| **모서리** | `rounded-2xl` (16px) | 부드러운 느낌 |
| **폰트** | Pretendard(본문) + Outfit(디스플레이) | 한영 조합 |
| **아이콘** | lucide-react | 일관된 스타일 |
| **애니메이션** | fade-in-up, float | 부드러운 진입 |
| **그라데이션** | `from-blue-500 to-cyan-400` (주), `from-blue-600 to-blue-500` (버튼) | 브랜드 시그니처 |

### 1.2 기존 컴포넌트 패턴

```
다크 정보 카드:   bg-white/3 border-white/5 rounded-xl p-4
라이트 카드:      bg-white border-gray-100 rounded-2xl p-7
글래스 카드:      glass-card rounded-2xl shadow-2xl shadow-black/20
배지:             px-2.5 py-0.5 rounded-md text-xs font-medium {color}/10
CTA 버튼:        bg-gradient-to-r from-blue-600 to-blue-500 rounded-xl
아이콘 박스:      w-10 h-10 rounded-lg bg-{color}-500/10
```

## 2. LMS 디자인 방향 결정

### 2.1 핵심 원칙

LMS는 매일 사용하는 **작업 환경**이므로, 기존 랜딩 페이지와 다른 접근이 필요합니다.

| 원칙 | 이유 | 적용 |
|------|------|------|
| **다크 테마 통일** | 장시간 사용 시 눈 피로 감소 + 기존 마이페이지와 일관성 | LMS 전체 다크 모드 |
| **사이드바 레이아웃** | LMS는 여러 메뉴를 오가는 앱형 구조 | 좌측 사이드바 + 우측 콘텐츠 |
| **정보 밀도 조절** | 대시보드는 한눈에, 상세는 여유 있게 | 카드 그리드 + 충분한 여백 |
| **진행 상태 시각화** | 학습 동기부여의 핵심 | 프로그레스 바, 원형 차트, 배지 |
| **마이크로 인터랙션** | 완료 체크, 호버, 전환 시 만족감 | 부드러운 transition + 체크 애니메이션 |

## 3. 색상 시스템

### 3.1 LMS 전용 색상 역할

기존 blue/cyan 팔레트를 유지하되, LMS 상태 표현용 색상을 체계화합니다.

| 역할 | 색상 | Tailwind | 용도 |
|------|------|----------|------|
| **브랜드/주요 액션** | 파랑 | `blue-500` / `blue-600` | 버튼, 링크, 활성 메뉴 |
| **진행률/성공** | 시안 → 그린 | `cyan-400` → `emerald-400` | 프로그레스 바, 완료 상태 |
| **경고/마감 임박** | 앰버 | `amber-400` / `amber-500` | 마감 D-3 이내, 주의 배지 |
| **위험/미제출** | 레드 | `red-400` / `red-500` | 미제출, 결석, 에러 |
| **멘토 관련** | 바이올렛 | `violet-400` / `violet-500` | 멘토 배지, 피드백 |
| **일반 텍스트** | 그레이 | `gray-400`(본문) `gray-500`(보조) `white`(강조) | 텍스트 계층 |
| **배경** | 다크 | `#0a0e1a`(기본) `#0f1420`(카드) `#131825`(사이드바) | 배경 계층 |

### 3.2 프로그레스 바 색상 규칙

```
  0% ~ 29%  →  red-500     (위험)
 30% ~ 59%  →  amber-500   (주의)
 60% ~ 79%  →  blue-500    (양호)
 80% ~ 100% →  emerald-500 (우수)
```

### 6.3 무료 컴포넌트 라이브러리 활용

| 라이브러리 | 용도 | 비용 | 비고 |
|-----------|------|------|------|
| **shadcn/ui** | 전체 UI 컴포넌트 | 무료 (MIT) | Tailwind 기반, 다크 모드 기본 지원 |
| **Recharts** | 차트 (진도율, 통계) | 무료 (MIT) | shadcn/ui의 Chart 컴포넌트와 호환 |
| **FullCalendar** | 세션 캘린더 | 무료 (MIT) | React 바인딩 제공 |
| **lucide-react** | 아이콘 | 무료 (ISC) | 기존 사용 중 |
| **tailwind-merge** | 클래스 병합 | 무료 | 기존 사용 중 |
| **clsx** | 조건부 클래스 | 무료 | 기존 사용 중 |

**추가 설치 권장:**
```bash
# shadcn/ui 초기화 (이미 clsx, tailwind-merge 설치되어 있으므로 호환)
npx shadcn-ui@latest init

# 필요 컴포넌트만 선택 설치
npx shadcn-ui@latest add card progress badge tabs avatar calendar tooltip

# 차트
npm install recharts

# 캘린더
npm install @fullcalendar/core @fullcalendar/react @fullcalendar/daygrid @fullcalendar/timegrid
```

> shadcn/ui는 컴포넌트를 프로젝트에 **복사**하는 방식이라 번들 사이즈 부담 없이 필요한 것만 사용할 수 있습니다. 기존 Tailwind + lucide-react 스택과 완벽하게 호환됩니다.

## 12. 팀 논의 사항

1. **shadcn/ui 도입 여부:** 기존 커스텀 컴포넌트에 shadcn/ui를 추가할지, 전체를 shadcn/ui로 통일할지
2. **차트 라이브러리:** Recharts vs Chart.js vs SVG 직접 구현
3. **사이드바 vs 탑 내비게이션:** 앱 느낌(사이드바) vs 웹 느낌(탑바)
4. **학습 스트릭 범위:** 최근 몇 주를 보여줄지, GitHub 잔디 스타일의 구체적 기준
