# AI 전체 RAG 평가 파이프라인 설계

## 목표

결정론적 골든 데이터셋을 확장하고 답변 근거성 지표를 추가하며 후보 승격 전후 평가를 자동 실행해 P3 전체 RAG 평가 파이프라인 항목을 완료합니다.

## 범위

- 기본 골든 데이터셋 범위를 200~500행으로 확장합니다.
- 기존 검색 재현율과 오래된 컨텍스트 지표를 유지합니다.
- 워크플로 의미 플래그, 필수 키워드, 금지 주장을 조합한 답변 근거성 지표를 추가합니다.
- 승격 전후 평가와 변화량 보고를 추가합니다.

## 접근 방식

기본 `load_dataset()`은 기존에 선별한 JSONL 행을 읽고 기본 평가 세트가 200행에 이를 때까지 해당 행에서 결정론적 범위 변형을 생성합니다. 원본 JSONL은 사람이 선별한 시드 말뭉치로 유지하며, 생성된 행에는 `generated_variant: true`를 표시합니다.

평가기에는 `answer_grounding_rate`를 추가합니다. 의미 위험 플래그(`evidence_missing`, `hallucination_suspected`, `contradiction_suspected`)가 없고, 설정된 필수 키워드를 모두 포함하며, 설정된 금지 주장을 피한 워크플로 답변을 근거가 있는 답변으로 판단합니다.

승격 워크플로는 승격 전과 재색인 후에 평가합니다. 보고서에는 주요 수치 지표의 `before_evaluation`, `evaluation`, `evaluation_delta`를 포함합니다.

## 실패 정책

평가는 기본적으로 결정론적 로컬 실행을 유지합니다. 린트나 재색인이 실패하거나 승격 후 검색 적중률이 임계값보다 낮으면 승격 워크플로는 실패합니다.

## 테스트

- 기본 데이터셋 로드가 고유한 행을 최소 200개 반환하는지 확인합니다.
- 평가기가 답변 근거성 지표를 보고하는지 확인합니다.
- 승격 워크플로가 승격 전 평가, 승격, 린트, 재색인, 승격 후 평가를 순서대로 호출하고 지표 변화량을 보고하는지 확인합니다.
