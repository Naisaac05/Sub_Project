# AI 리뷰 Ollama 속도 설계

> 목표: 답변 품질을 위해 `qwen3:4b-q4_K_M`을 유지하면서 모델 작업량을 줄여 로컬 CPU 지연 시간을 단축합니다.

## 문제

AI 리뷰 흐름은 짧은 학습 지도 답변을 사용하지만 Ollama 프롬프트에는 모델이 필요로 하는 것보다 많은 컨텍스트가 포함됩니다. 4코어 CPU 노트북에서는 이로 인해 첫 토큰과 전체 응답 시간이 늘어납니다. 모델 크기를 줄이면 속도는 향상되지만 답변 품질이 저하될 수 있습니다.

## 전략

기존 백엔드를 신뢰 가능한 원천으로 사용하고 Qwen은 응답 표현만 담당하게 합니다.

- Qwen 4B를 기본 모델로 유지합니다.
- 질문, 선택한 답, 정답, 학습자 답변, 규칙 평가만 전송합니다.
- 필요한 경우가 아니면 전체 선택지 목록을 전송하지 않습니다.
- 백엔드의 사실 정보가 권위 있는 정보임을 모델에 알립니다.
- 출력을 짧은 한국어 문장 2~3개로 제한합니다.
- 동일한 Ollama 프롬프트를 메모리에 캐시해 반복 자유 질문이나 재시도에서 모델을 다시 호출하지 않습니다.
- 4B 모델을 유지하면서 출력 토큰과 컨텍스트 창을 줄이는 등 4코어 CPU에 맞게 기본값을 조정합니다.

## 제외 범위

- OpenAI를 사용하지 않습니다.
- DB 스키마를 변경하지 않습니다.
- Redis 캐시를 사용하지 않습니다.
- 더 작은 모델을 기본값으로 사용하지 않습니다.

## 인수 조건

- 기본 제공자는 Ollama로 유지됩니다.
- 기본 모델은 `qwen3:4b-q4_K_M`으로 유지됩니다.
- 백엔드가 컴파일됩니다.
- 반복되는 동일 프롬프트는 캐시에서 응답합니다.
- 프롬프트 크기가 이전 전체 컨텍스트 프롬프트보다 명확히 작습니다.
