# AI 리뷰 오타·별칭 해석기 설계

## 목표

Phase 4.9에서는 학습자의 자유 질문에 경량 오타·별칭 해석기를 추가합니다. 작은 철자 오류나 대체 명칭이 포함돼도 올바른 빠른 경로 답변과 RAG 개념 카드에 도달하도록 합니다.

## 범위

해석기는 Python AI 서버 안에서만 실행됩니다. LLM을 호출하거나 Spring/Next API 계약을 변경하지 않으며, 새 지식을 자동 승격하지도 않습니다. 경량 답변 계층이나 승인된 개념 카드가 이미 알고 있는 신뢰도 높은 개념 용어에만 집중합니다.

## 아키텍처

전용 `app.workflow.query_resolver` 모듈을 추가합니다. 이 모듈은 학습자 질문을 받아 표준 개념 용어, 별칭, 알려진 오타 변형의 소규모 레지스트리와 비교한 뒤, 보정된 질문·개념 ID·일치 용어·보정 유형·신뢰도를 담은 `ResolvedQuery` 객체를 반환합니다.

워크플로는 의도 분류, 빠른 경로 답변 조회, RAG 검색 전에 보정된 질문을 사용합니다. 신뢰도가 높은 오타·별칭 일치는 내부적으로 다시 쓰고, 신뢰도가 낮거나 알 수 없는 텍스트는 변경하지 않습니다.

## 일치 정책

1. 정확히 일치하거나 정규화된 별칭을 가장 먼저 선택합니다.
2. 하나의 명확한 표준 용어에 대응하는 알려진 오타 변형을 허용합니다.
3. 일반 퍼지 매칭은 보수적으로 적용합니다. 길이가 긴 라틴 문자 토큰만 비교하고 강한 일치만 허용합니다.
4. 모호하거나 약한 일치는 원래 질문을 유지합니다.

## 데이터 흐름

1. 사용자가 자유 질문을 입력합니다.
2. `resolve_learner_query()`가 원래 질문 또는 보정된 질문을 반환합니다.
3. `retrieve_context_node()`가 해석기 메타데이터를 워크플로 상태에 저장합니다.
4. 의도 분류와 RAG 질의가 보정된 질문을 사용합니다.
5. `generate_answer_node()`가 보정된 질문을 `lightweight_answer_for()`에 전달합니다.

## 테스트

정확한 별칭, 알려진 오타 보정, 약한 일치 거부, 보정 후 RAG 검색, 보정 후 빠른 경로 동작을 테스트합니다.
