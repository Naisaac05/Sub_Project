# EXAONE 2.4b 적용 가이드 (follow-up 모델 교체)

본 가이드는 [REPORT.md](REPORT.md)의 권장사항을 팀원이 직접 적용할 수 있도록 정리한 절차 문서입니다.

- **변경 대상**: AI 복습 전체 생성 모델 + follow-up/fallback + Adaptive Judge 모델
- **변경 내용**: 기본 런타임 모델을 `exaone3.5:2.4b`로 통일
- **코드 수정**: 기본값 반영 완료. 환경변수는 명시적으로 덮어쓸 때만 사용
- **소요 시간**: 약 5~10분 (모델 다운로드 포함)
- **환원**: 환경변수 되돌리면 즉시 (1분)

---

## 사전 이해 — 왜 환경변수 하나로 끝나는가

[ai/app/ollama/client.py:20](../../app/ollama/client.py#L20):
```python
FALLBACK_MODEL = os.getenv("PYTHON_AI_FALLBACK_MODEL", "exaone3.5:2.4b")
```

[ai/app/workflow/nodes.py:434-437](../../app/workflow/nodes.py#L434):
```python
def _generation_model_for_state(state):
    if state.mode == "follow-up":
        return FALLBACK_MODEL   # ← 환경변수로 제어
    return state.request.model
```

기본값은 이미 EXAONE 2.4b입니다. 필요하면 `PYTHON_AI_MODEL`, `PYTHON_AI_FALLBACK_MODEL`, `PYTHON_AI_JUDGE_MODEL`로 명시적으로 덮어쓸 수 있습니다.

  ```powershell

  ```
  출력에 `exaone3.5:2.4b` 가 보여야 정상.

## Step 2. 환경변수 설정 (3가지 방법 중 택1)

### 방법 A: `.env` 파일 사용 ⭐ 팀 표준 권장

`.env` 파일이 있는지 확인:

```powershell
ls ai/.env
```

**있으면** 파일에 다음 줄 추가 (또는 기존 줄 수정):
```
PYTHON_AI_FALLBACK_MODEL=exaone3.5:2.4b
```

**없으면** `ai/.env` 새로 만들고 위 줄만 작성하면 됨.

### 방법 B: PowerShell 세션에서 임시 (개인 테스트용)

```powershell
$env:PYTHON_AI_FALLBACK_MODEL="exaone3.5:2.4b"
```

이 PowerShell 창에서만 유효. 창 닫으면 사라짐. **운영용 아님 — 빠른 시도용**.

### 방법 C: Windows 시스템 환경변수 (영구)

1. `Win + R` → `sysdm.cpl` → Enter
2. **고급 탭** → **환경 변수** 클릭
3. 사용자 변수 영역에서 **새로 만들기**
4. 변수 이름: `PYTHON_AI_FALLBACK_MODEL`
5. 변수 값: `exaone3.5:2.4b`
6. 확인 → 확인
7. PowerShell 새 창 띄워야 적용됨

## Step 3. Python AI 서비스 재시작

기존 uvicorn이 돌고 있다면:

1. 해당 PowerShell 창에서 **Ctrl + C** 로 중지
2. 재실행:
   ```powershell
   cd C:\Users\aucu2\Sub_Project
   .\ai\.venv\Scripts\Activate.ps1
   cd ai
   uvicorn app.main:app --host 127.0.0.1 --port 8001 --reload
   ```

⚠️ `--reload` 는 **코드 변경**에만 자동 반영. **환경변수 변경은 반드시 재시작** 필요.

## 자주 묻는 질문 (FAQ)

### Q1. `PYTHON_AI_MODEL` 과 `PYTHON_AI_FALLBACK_MODEL` 의 차이?

| 환경변수 | 영향 범위 | 기본값 |
|---|---|---|
| `PYTHON_AI_MODEL` | first-question, free-question 1차 호출 | `exaone3.5:2.4b` |
| `PYTHON_AI_FALLBACK_MODEL` | **follow-up 직접 사용** + 다른 모드의 폴백 | `exaone3.5:2.4b` |
| `PYTHON_AI_JUDGE_MODEL` | Adaptive Judge | `exaone3.5:2.4b` |

현재 권장은 세 모델 경로를 모두 `exaone3.5:2.4b`로 통일하는 것입니다.

### Q2. 디스크 용량 부담은?
- EXAONE 2.4b: 1.6GB 추가
- 기존 `qwen3:4b-q4_K_M` (2.5GB) 는 **삭제하지 마세요** — 롤백 시 필요
- 한참 안정화 후 디스크 회수 원하면:
  ```powershell
  ollama list
  ```

### Q3. 다른 모드(first-question, free-question)에 영향?
- `follow-up` 모드: 100% 영향 (직접 사용)
- 다른 모드: **폴백 경로 활성화 시에만** 영향 (1차 호출 실패 시)
- 평시엔 `PYTHON_AI_MODEL`도 `exaone3.5:2.4b`를 사용

### Q4. CPU only 환경에서 EXAONE 7.8b 는 안 되나요?
- 1~2 tok/s 로 스트리밍 UX 불가 (한 응답 30~60초+)
- GPU 서버 환경이면 검토 가치 있음. 그 경우 [REPORT.md](REPORT.md) 한계 절 참고하여 추가 실험 권장.

### Q5. 운영 배포는?
- 본 가이드는 로컬/dev 환경 기준
- 운영 배포 시 컨테이너/서비스 환경변수 설정 방식에 맞춰 적용
- Rolling deploy 권장 — 한 인스턴스 먼저 적용, 로그 모니터링 후 전체 적용

### Q6. 변경 후 응답 품질 개선이 안 느껴진다면?
1. `ollama list` 로 모델 다운로드 확인
2. 서비스 재시작 했는지 확인
3. 응답 JSON의 `model_used` 필드 확인
4. 캐시 영향 가능성 — `model_used` 끝에 `:cache` 붙어 있으면 이전 모델의 캐시된 응답
5. 같은 입력으로 여러 번 호출해보고 차이 확인
