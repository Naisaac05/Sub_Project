# 프로젝트 규칙

## 에러 로깅 규칙

이 프로젝트에서 에러/버그를 원인까지 밝혀 해결했을 때는 **항상** `error/` 폴더에 기록을 남깁니다.

### 언제 기록하나

- 원인을 실제로 특정하고 수정/우회 조치를 취한 경우
- 겉보기에는 해결됐지만 근본 원인이 남아 재발 여지가 있는 경우 (메모 필수)
- 팀원이 다시 만날 가능성이 있는 환경/세팅/시드 데이터 이슈

아래는 기록하지 않아도 됩니다:

- 오타/단순 컴파일 에러처럼 로그로 남길 가치가 없는 것
- 원인 파악 없이 재시도만으로 해결된 일시적 네트워크/프로세스 문제

### 어떻게 기록하나

1. 파일명: `error/YYYY-MM-DD-짧은-제목.md`
2. 내용 구조는 `error/README.md`의 템플릿을 그대로 사용합니다 (증상 / 원인 / 해결 방법 / 재발 방지·메모).
3. 관련 파일은 `경로:line` 형식으로 정확히 링크합니다.
4. 작성 후 `error/README.md`의 "인덱스" 섹션에 한 줄로 링크를 추가합니다.

### 언제 수행하나

- 에러를 해결한 **그 턴에서** 바로 작성합니다. "나중에 몰아서" 하지 않습니다.
- 해결 완료 보고와 함께 어떤 파일에 기록했는지도 같이 알려주세요.
